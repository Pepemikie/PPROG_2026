Implementación de F7: Modificación del módulo Object para incorporar nuevas propiedades.

Cambios realizados:

1. En include/object.h:
   - Agregadas declaraciones de funciones para las nuevas propiedades: object_set_health, object_get_health, object_set_movable, object_get_movable, object_set_dependency, object_get_dependency, object_set_open, object_get_open.

2. En src/object.c:
   - Modificada la estructura _Object para incluir los nuevos campos: int health, Bool movable, Id dependency, Id open.
   - Inicializados los nuevos campos en object_create: health=0, movable=FALSE, dependency=NO_ID, open=NO_ID.
   - Implementadas las funciones get/set para cada nueva propiedad.
   - Actualizada object_print para mostrar los nuevos campos en DEBUG.

3. En include/player.h y src/player.c:
   - Agregada función player_modify_health para modificar la salud del jugador sumando/restando puntos.

4. En include/game.h y src/game.c:
   - Agregada función game_get_link para obtener un enlace por su id.

5. En src/game_actions.c:
   - Modificada game_actions_take para:
     - Verificar que el objeto sea movable antes de cogerlo.
     - Verificar que el objeto de dependencia esté en el inventario si aplica.
     - Modificar la salud del jugador al coger el objeto.
     - Abrir el enlace correspondiente si el objeto tiene open != NO_ID.
   - Modificada game_actions_drop para dejar también todos los objetos que dependan del objeto dejado.

6. En src/game_reader.c:
   - Actualizado el parsing de objetos para leer los nuevos campos desde archivos de datos: health|movable|dependency|open.

Estos cambios permiten que los objetos tengan propiedades avanzadas como salud, movilidad, dependencias y capacidad de abrir enlaces, integrando estas funcionalidades en la lógica del juego.
